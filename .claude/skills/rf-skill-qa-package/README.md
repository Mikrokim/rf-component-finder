# חבילת סקיל rf-component-search + מבחני QA

## תוכן התיקייה

```
rf-skill-qa-package/
├── README.md                        ← הקובץ הזה
├── rf-component-search/
│   ├── SKILL.md                     ← הסקיל המלא (גרסה מעודכנת, קובץ יחיד עצמאי)
│   └── vendors.md                   ← עותק ייחוס של רשימות הספקים (מוטמע גם ב-SKILL.md)
└── qa-tests/
    ├── test-1-amplifier-main-path.md
    ├── test-2-impossible-spec-honesty.md
```

## התקנת הסקיל

1. כווצו את התיקייה `rf-component-search` ל־ZIP (קליק ימני ← Send to ← Compressed (zipped) folder).
2. ב־claude.ai: Settings ← Capabilities ← ודאו ש־Code execution מופעל ← Skills ← Upload skill ← בחרו את ה־ZIP.
3. אם מותקנת גרסה קודמת — הסירו אותה קודם.

## הרצת המבחנים (הנחיות ל־QA)

- כל מבחן בשיחה חדשה ונקייה (לא באותה שיחה).
- ודאו שהסקיל הופעל (מופיע אינדיקטור שימוש בסקיל בתשובה).
- כל קובץ מבחן מכיל: הפרומפט להעתקה, קריטריוני עובר, קריטריוני נכשל.
- שמרו את קבצי ה־Excel שנוצרו ואת הטקסט המלא של כל תשובה.
- תעדו כל חריגה, גם קטנה — כולל צילומי מסך.

## מה הסקיל אמור לעשות (תקציר)

חיפוש פרמטרי של רכיבי RF: פירוק מפרט ← שאלות הבהרה ← חיפוש רחב עם החרגת 12 אתרי היצרנים הקבועים ← סריקת רשימת יצרנים ייעודית ← גל חיפושים נגזר ← אימות כל פרמטר מול datasheet של היצרן ← אימות חוזר עצמאי ← דוח: טבלה בצ'אט + Excel עם 3 גיליונות (התאמות / נבדקו־ונפסלו / יומן כיסוי) + הצהרת כיסוי כנה.

## מה השתנה בגרסה הזו

שני חיזוקים לשכבת האמון של הסקיל:

1. **snippet מקדם, לא פוסל (Step 3 + Efficiency + יומן כיסוי):** תוצאת חיפוש/snippet יכולה רק להוסיף מועמד לתור בדיקת ה־datasheet — לעולם לא לפסול אותו. פסילה מתבצעת אך ורק מול ערך datasheet בפועל. מועמד שנזרק על סמך snippet בלבד (רק כשהמאגר גדול מדי) מתועד ביומן הכיסוי כ־*snippet-filtered (not datasheet-verified)*, כדי שחיפוש יסודי יהיה נבדל מחיפוש שגזם בזול. זה סוגר את הפער של "לא נפספס התאמה קיימת".

2. **שני מצבי אימות חוזר, כולל מצב מוקשח בלי subagents (Step 3.5):** זמינות subagents תלויה בסביבת הריצה ולא במודל — לכן הסקיל בוחר מצב במפורש ומצהיר עליו. מצב subagent (מועדף) = הפרדה אמיתית. מצב single-agent מוקשח (הנפוץ, כולל מודל חזק בצ'אט רגיל) = טבלה שנייה ריקה בלי הצצה לראשונה + ציטוט מיקום מדויק לכל ערך + הורדת ביטחון (✅ מעורפל → ⚠️). מצב האימות מדווח בהצהרת הכיסוי.
