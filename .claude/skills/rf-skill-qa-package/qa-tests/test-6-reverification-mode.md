# מבחן 6 — מצב אימות חוזר (Step 3.5)

**מטרה:** לוודא שהסקיל בוחר במפורש את מצב האימות החוזר, מצהיר עליו, ומיישם את המצב המוקשח (single-agent) כמו שצריך כשאין subagents.

**רקע:** זמינות subagents תלויה בסביבת הריצה ולא במודל — אותו מודל חזק יכול לרוץ עם היכולת (סביבה agentic) או בלעדיה (צ'אט רגיל). הסקיל לא אמור להניח נוכחות subagents. במצב single-agent המוקשח נדרשים: טבלה שנייה ריקה בלי הצצה לראשונה, ציטוט מיקום מדויק לכל ערך, וערך בלי ציטוט נחשב לא־מאומת.

**הוראות:** להריץ בשיחה חדשה ונקייה. לוודא שהסקיל rf-component-search הופעל. המפרט הזה אמור להניב לפחות התאמה אחת או שתיים, כדי שיהיה על מה להפעיל את שלב האימות החוזר.

## הפרומפט (להעתקה)

```
find an amplifier 8-12 GHz, gain 20-28db, OP1dB 20dbm min, NF 4.5db max
```

## קריטריוני עובר

- [ ] הצהרת הכיסוי מציינת במפורש באיזה מצב אימות חוזר נעשה שימוש (subagent / single-agent)
- [ ] לא הניח שיש subagents רק כי המודל חזק — הבחירה מבוססת על היכולת בפועל בסביבה
- [ ] אם רץ במצב single-agent: לכל ערך של רכיב ✅/⚠️ מופיע ציטוט מיקום מ־ה־datasheet (עמוד / טבלה / שורה)
- [ ] ערך שלא נמצא לו ציטוט מיקום סומן כלא־מאומת (⚠️ או "requires manufacturer contact"), לא כמאומת
- [ ] הופעלו בדיקות השפיות הפיזיקליות (למשל OIP3 מול OP1dB) והופיע תיעוד אם משהו הצריך בדיקה חוזרת
- [ ] במצב single-agent: ✅ שנשען על קריאה בודדת ומעורפלת הורד ל־⚠️

## קריטריוני נכשל

- לא ציין כלל באיזה מצב אימות רץ
- הציג ✅ בלי ציטוטי מיקום במצב single-agent
- הצהיר "אומת על ידי subagent" בסביבה שבה אין subagents (אישור־בעיוורון מדומה)
- דילג על שלב האימות החוזר לגמרי

## מה לשמור

הטקסט המלא של התשובה, קובץ ה־Excel, וצילום של הצהרת הכיסוי.
