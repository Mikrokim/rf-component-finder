# מבחן 4 — מפרט עמום (בדיקת עצירה)

**מטרה:** לוודא שכשהמפרט לא חד־משמעי, הסקיל עוצר ושואל — ולא רץ לחפש עם הנחות שקטות. הנחה שגויה אחת (למשל P1dB כניסה במקום מוצא) פוסלת את כל תוצאות החיפוש.

**הוראות:** להריץ בשיחה חדשה ונקייה. לוודא שהסקיל rf-component-search הופעל.

## הפרומפט (להעתקה)

```
מגבר 14-15GHz gain 20 P1dB 24
```

## קריטריוני עובר

- [ ] עצר ושאל שאלות הבהרה לפני שביצע ולו חיפוש אחד
- [ ] השאלות מכסות לפחות: P1dB בכניסה או במוצא? gain 20 — מינימום או ערך נדרש מדויק? האם יש דרישות נוספות (NF, OIP3, מתח הזנה, צורת רכיב)?
- [ ] השאלות מרוכזות בסבב אחד (לא טפטוף של שאלה־תשובה־שאלה)
- [ ] אם הניח הנחת ברירת מחדל (למשל OP1dB במוצא) — הצהיר עליה במפורש
- [ ] רק אחרי קבלת תשובות — התחיל את החיפוש

## קריטריוני נכשל

- התחיל לחפש מיד בלי לשאול
- הניח הנחות בלי להצהיר עליהן
- שאל שאלות אבל התחיל לחפש במקביל בלי לחכות לתשובות

## תרחיש המשך (אופציונלי, מומלץ)

לאחר השאלות, לענות: "P1dB במוצא, gain מינימום, NF עד 5, בלי דרישות נוספות" — ולוודא שהחיפוש שממשיך משקף בדיוק את התשובות (למשל: לא נפסל מגבר עם gain 28).

## מה לשמור

צילום מסך של שאלות ההבהרה, הטקסט המלא של השיחה.
